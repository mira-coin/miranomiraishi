# Codex実装タスク

`GAME_DESIGN.md`を読み、ブラウザで遊べるMVPを実装してください。

## 最優先
- まず3D素材なしでも遊べる完成状態を作る
- Three.jsのBoxGeometry等で3部屋を構成
- 後からGLBと画像を差し替えられる設計
- 進行不能を絶対に作らない
- 想定プレイ時間は初見15〜30分
- 日本語・英語・インドネシア語を最初に選べ、ゲーム中も設定から変更できる

## MVP仕様
- 一人称3D探索
- WASD / 矢印キー移動
- マウス視点
- Eキーまたはクリックで調査
- スマホ用仮想スティックと調査ボタン
- 右側にミラUI
- 調査 / 解析 / 未来視 / 記録ボタン
- 未来視クールダウン45秒
- 3部屋、3種類の謎
- タイトル画面
- クリア画面
- localStorageセーブ
- BGM、SEのON/OFF
- 初回起動時に言語選択画面を表示
- 言語選択は日本語 / English / Bahasa Indonesia
- 選択言語はlocalStorageに保存
- 設定画面からいつでも言語変更可能
- 言語変更後は再読み込みなしでUIと会話を即時反映
- セーブデータや謎の進行状態は言語変更で初期化しない

## ボリューム設計
- 初見クリア15〜30分
- 慣れたプレイヤーは10〜15分程度
- 3部屋構成
- 大謎3個
- 小謎4〜6個
- 取得アイテム6〜8個
- 必須会話30本以上
- 調査テキスト25本以上
- ヒントは各大謎につき3段階
- 通常エンド1つ、可能なら追加ログ回収で真エンド1つ
- 無意味な移動時間や同じ場所の往復で水増ししない

## 謎
### Room 1 観測の部屋
監視カメラを向けている間だけ壁記号が変化する。3方向から観測した記号を重ねると解除コードになる。

### Room 2 音のない暗証番号
機械音の間隔を4つ読み取り、順序を部屋内の点滅ランプから決める。ミラ解析は波形を表示するが答えは直接出さない。

### Room 3 未来の残像
移動床の5秒後の位置が残像で見える。未来視を使うと残像が見えるが、プレイヤーの行動で一部が変化する。固定される1つの足場だけを見抜く。

## 多言語設計
- 文字列をHTMLやJavaScriptへ直接書かない
- すべて翻訳キーで管理する
- UI、会話、調査文、ヒント、アイテム名、アイテム説明、目的表示、エラー、クリア画面を3言語対応する
- 固有名詞は以下で統一する
  - ミラ: Mira
  - ヨウ: You
  - ミラの未来視: Mira's Foresight / Penglihatan Masa Depan Mira
  - 囚われた電脳の迷宮: The Captive Cyber Labyrinth / Labirin Siber yang Menawan
- 英語とインドネシア語で文字が長くなってもUIが崩れない可変レイアウトにする
- ボタン幅固定による文字切れを禁止
- フォントサイズ縮小だけで解決せず、折り返しと可変高さに対応する
- 翻訳不足時は日本語ではなく英語へフォールバックする
- 開発用に未翻訳キーを検出する警告をconsoleへ出す

## UI
- ミラ立ち絵は仮PNGでもよい
- 表情状態: normal / smug / worried / serious / smile
- 会話はJSON管理
- 二重表示、連打、イベント重複を防止
- 言語ボタンは国旗だけにせず文字でも表示する
- タイトル画面と設定画面の両方に言語変更を置く
- 字幕速度を3段階で変更可能にする
- 自動送りON/OFF、ログ表示、文字サイズ変更を可能にする

## 掛け合い
最低30本。短くテンポよく。
ミラ一人称は「ミラ」または「わたし」。俺・おまえは禁止。
3言語とも性格と距離感を維持し、直訳で不自然にならないようにする。

## 安全設計
- 各部屋にリセットボタン
- セーブ破損時は初期化
- 重要アイテム紛失なし
- ドア開放後は再ロックしない
- 会話中に移動しても進行が壊れない
- 言語切替中に会話イベントや謎判定が重複しない
- 未翻訳文字列があってもゲーム進行を止めない
