# Codex実装タスク

`GAME_DESIGN.md`を読み、ブラウザで遊べるMVPを実装してください。

## 最優先
- まず3D素材なしでも遊べる完成状態を作る
- Three.jsのBoxGeometry等で3部屋を構成
- 後からGLBと画像を差し替えられる設計
- 進行不能を絶対に作らない

## MVP仕様
- 一人称3D探索
- WASD / 矢印キー移動
- マウス視点
- Eキーまたはクリックで調査
- スマホ用仮想スティックと調査ボタン
- 右側にミラUI
- 調査 / 解析 / 未来視 / 記録ボタン
- 未来視クールダウン45秒
- 3部屋、3種類の謎
- タイトル画面
- クリア画面
- localStorageセーブ
- BGM、SEのON/OFF

## 謎
### Room 1 観測の部屋
監視カメラを向けている間だけ壁記号が変化する。3方向から観測した記号を重ねると解除コードになる。

### Room 2 音のない暗証番号
機械音の間隔を4つ読み取り、順序を部屋内の点滅ランプから決める。ミラ解析は波形を表示するが答えは直接出さない。

### Room 3 未来の残像
移動床の5秒後の位置が残像で見える。未来視を使うと残像が見えるが、プレイヤーの行動で一部が変化する。固定される1つの足場だけを見抜く。

## UI
- ミラ立ち絵は仮PNGでもよい
- 表情状態: normal / smug / worried / serious / smile
- 会話はJSON管理
- 二重表示、連打、イベント重複を防止

## 掛け合い
最低20本。短くテンポよく。
ミラ一人称は「ミラ」または「わたし」。俺・おまえは禁止。

## 安全設計
- 各部屋にリセットボタン
- セーブ破損時は初期化
- 重要アイテム紛失なし
- ドア開放後は再ロックしない
- 会話中に移動しても進行が壊れない
